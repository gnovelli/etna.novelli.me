Executive Summary

Progetto ETNA - Data: 27 giugno 2025

1. Introduzione

Il progetto ETNA è una piattaforma software multi-parametrica per il monitoraggio dell'attività vulcanica del Monte Etna. Integra dati geofisici e geochimici, li elabora e li analizza con tecniche di machine learning per individuare possibili anomalie, generando report tramite agenti basati su LLM. L'architettura è containerizzata per favorire la modularità e il deployment.

2. Scopo e Obiettivi del Progetto

L'obiettivo è supportare vulcanologi e protezione civile nel monitoraggio del vulcano attraverso:

  • Acquisizione dati centralizzata da stazioni sismiche, infrasoniche e da fonti di degassamento.
  • Elaborazione dei dati con calcolo di parametri come SSAM, polarizzazione e correlazione incrociata.
  • Rilevamento di anomalie con modelli di deep learning (Autoencoder Convoluzionali).
  • Reporting assistito tramite agenti che utilizzano LLM via Ollama.
  • Visualizzazione web e notifiche via Telegram.

3. Architettura del Sistema

Il sistema adotta un'architettura a microservizi orchestrata con Docker Compose. I principali componenti includono:

  • Data Ingestion con obspy e server FDSN configurati in config/stations.yaml.
  • Pipeline di elaborazione con calcolo di SSAM, RMS, analisi di polarizzazione e correlazione.
  • Un Autoencoder Convoluzionale per il rilevamento di anomalie nei dati.
  • Scheduler che pianifica l'acquisizione, l'elaborazione e il reporting.
  • Agenti autonomi per la generazione di analisi tramite LLM.
  • API RESTful basata su FastAPI e dashboard interattiva con Streamlit.
  • Modulo di notifica via Telegram.

4. Stack Tecnologico

Il progetto utilizza Python 3 e librerie come obspy, pandas, torch e FastAPI. La dashboard è realizzata con Streamlit e Plotly, mentre il deployment avviene tramite Docker e Docker Compose. Nginx funge da reverse proxy e GitHub Actions gestisce il processo di CI/CD.

5. Deployment e Configurazione

Il sistema è eseguito tramite Docker. Il file docker-compose.yml definisce i servizi, mentre config/stations.yaml gestisce le stazioni di monitoraggio. Il workflow di GitHub Actions in .github/workflows/deploy.yml automatizza build e deploy.

6. Conclusioni

ETNA combina analisi geofisiche e intelligenza artificiale per un monitoraggio continuo e accessibile. La dashboard rende i dati interpretabili, mentre gli agenti LLM offrono un approccio innovativo alla sintesi dei risultati.

7. ETNA GPT Expert

Il progetto include un assistente "ETNA GPT Expert" progettato per rispondere alle domande sugli output, guidare all'uso del sistema e aiutare gli sviluppatori nella comprensione della codebase.
